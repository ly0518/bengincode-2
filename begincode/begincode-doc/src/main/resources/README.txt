版本号命名规则
版本号共三位，依次为主版本号、次版本号、小版本号。版本号变更原则
1、版本号与工程版本一致
2、如数据库字段变更，数据字典修改，需要修改小版本号
3、如增加表，以满足原有功能的需求，应修改次版本号
4、如有重大修改，如新增模块，修改主版本号

创建升级脚本步骤
1、在ddl下创建版本文件夹，命名为新版本号，在版本文件夹下创建increment文件夹
2、创建升级脚本，参照SQL脚本命名规范。
   必须要有全量脚本和增量脚本，增量脚本放到increment下，全量脚本放到版本文件夹下。
   保证全量脚本和增量脚本的执行效果一致。
3、修改insert_db_version.sql中的版本号
4、修改工程版本号，父pom的version属性以及所有子工程的version
5、提交代码

数据库升级步骤
1、更新代码
2、检查当前版本是否是最新版本，如本地版本是1.0.0，而在ddl下有新文件夹名称为1.0.1,说明本地数据库需要更新
3、依次执行alter前缀的sql（次序由脚本名称可知）
4、依次执行data前缀的sql
5、执行insert_db_version.sql

若本地版本与最新版差别较大，可以直接干掉本地数据库，执行最新版全量脚本
